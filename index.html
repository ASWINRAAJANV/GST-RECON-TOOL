Capabilities

Everything consolidated.
Nothing manual.

Eliminates the repetitive, error-prone work of consolidating GST return data and performing monthly reconciliations.

📥

Reads Directly from GST Portal

✅ What This Tool Does
  • ✅ Parses GSTR-1, GSTR-3B, CMP-08 PDFs for any Financial Year
  • ✅ Consolidates GSTR-2B and GSTR-4A Excel files and produces ITC summary sheets
  • ✅ Generates Turnover Recon, ITC Recon, and E-Credit Ledger summary workbook
  • ✅ Extracts and aggregates HSN summary (Section 12) data
  • ✅ GST-standard Excel formatting: colour coding, freeze panes, tab colours
  • ✅ GSTR-1A amendment return support with automatic merging
  • ✅ Session-aware — remembers GSTIN, FY, and folder paths per client
  • ✅ Dual logger: clean user.log + full debug.log for diagnostics
❌ What This Tool Does NOT Do
  • ❌ Does not file returns or connect to the GST Portal in any way
  • ❌ Does not transmit any data to any server, cloud, or third party
  • ❌ Does not require an internet connection during processing
  • ❌ Does not require Python or any other software to be pre-installed
  • ❌ Does not collect any analytics or telemetry of any kind
  • ❌ Does not store client data in any remote location

🛡️ DPDP Act 2023 Compliant — All processing happens entirely on your local computer. Your GSTIN, client information, return data, and output files never leave your machine.

How It Works

Three ways to run.
One correct answer each time.

Recommended
A
Reconciliation Engine

Runs the complete Regular scheme pipeline in one go — GSTR-1, GSTR-3B and GSTR-2B Mirror Engines followed by the GST Reconciliation workbook. Select your input folders, click Run.

Best for
Regular scheme taxpayers doing month-end or year-end processing when all return files are available
B
Mirror Engines

Run engines individually or all-at-once per scheme. Regular scheme: GSTR-1, GSTR-3B, GSTR-2B. Composition scheme: CMP-08, GSTR-4A. Each scheme has its own "Run All" button.

Best for
Composition dealers, or re-running one engine after adding a missing month without reprocessing everything
C
Reconciliation Only

All consolidated files already exist? Skip the mirrors and only regenerate the Reconciliation Workbook. Saves time when the source data hasn't changed.

Best for
Regenerating the Recon Workbook without re-processing all return PDFs
Input Files

What you need to provide.

Download these directly from the GST Portal. No manual formatting required before loading into the tool.

📄 GSTR-1 PDFs PDF
Portal path: Services → Returns → Returns Dashboard → Select Period → GSTR-1 → View Summary → Download PDF
One official filed return PDF per month (Apr–Mar). All months in one folder. Any filename is fine — the tool reads GSTIN and period from inside the PDF.
📄 GSTR-3B PDFs PDF
Portal path: Services → Returns → Returns Dashboard → Select Period → GSTR-3B → Download PDF
Same requirements as GSTR-1 — one PDF per month, all months in one folder.
📊 GSTR-2B Excel Files XLSX
Portal path: Services → Returns → Returns Dashboard → Select Period → GSTR-2B → View → Download GSTR 2B Details (Excel)
One file per month/quarter. Do not open and re-save in Excel before processing.
📄 GSTR-1A Amendment PDFs PDF · Optional
Collect GSTR-1A PDFs in a separate folder. Enable the GSTR-1A toggle in the engine screen and browse to that folder. The engine merges them automatically.
📄 CMP-08 PDFs PDF · Composition
Portal path: Services → Returns → Returns Dashboard → Select Period → View CMP-08 → Download
One PDF per Quarter (Apr–Mar). All quarters for the FY in one folder.
📊 GSTR-4A Excel XLSX · Composition
Portal path: Services → Returns → Returns Dashboard → Select Period → GSTR-4A → Download
One Excel file per quarter. All quarters in a single folder. Do not re-save before use.
⚠️ Important: Do NOT mix files from different GSTINs or different Financial Years in the same input folder. This will cause incorrect output or processing errors.
